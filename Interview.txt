Interview Questions
Q1. Why use a monorepo?

Answer

Shared packages
Easier dependency management
Atomic commits
Faster CI/CD
Common at large organizations
Q2. Why PNPM instead of NPM?

Answer

Faster installs
Disk-space efficient
Better monorepo support
Symlinked dependency graph
Q3. Why TurboRepo?

Answer

Incremental builds
Task caching
Faster CI
Scales well to many packages
Q4. Why separate core from client?

Answer

Framework logic stays independent
Reusable in multiple renderers
Easier testing
Similar to how React separates core and DOM renderer

Interview Questions
Q: SSR vs CSR?
Answer

SSR:

Better SEO
Faster first paint
Better performance on slow devices

CSR:

Rich interactivity
Faster navigation after initial load
Reduced server work

Modern systems combine both.


Interview Questions
Q: Why Redis adapter in Socket.IO?
Answer

In distributed systems:

Multiple server instances exist
Socket connections are distributed

Redis adapter synchronizes events across all instances.

Interview Questions
Q: Explain hydration mismatch.
Answer

Hydration mismatch occurs when:

Server-rendered HTML differs from client-rendered HTML.

Causes:

Non-deterministic rendering
Date/random values
Browser-only APIs

Fix:

Stable rendering logic
Avoid random values during SSR
Q: Explain concurrent rendering.
Answer

Concurrent rendering allows React-like systems to:

Pause rendering
Resume rendering
Prioritize important updates

Improves responsiveness.

Q: Why use message queues?
Answer

Queues:

Decouple services
Improve reliability
Handle retries
Smooth traffic spikes
Enable async processing
Q: Explain eventual consistency.
Answer

In distributed systems:

Data may temporarily differ
Eventually all replicas synchronize

Common in:

Caching
Replication
Event-driven systems
